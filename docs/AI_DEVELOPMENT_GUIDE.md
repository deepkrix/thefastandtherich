# TheFastAndTheRich
# AI DEVELOPMENT GUIDE

Version: 1.0  
Status: Living Document  
Document Type: Artificial Intelligence Development Guidelines

---

# 1. Purpose

This document defines how artificial intelligence systems should support development of TheFastAndTheRich.

It describes:

- AI development rules
- AI responsibilities
- architectural constraints
- code generation standards
- documentation requirements
- review processes

This document ensures that AI-assisted development remains consistent with the project architecture.

---

# 2. AI Development Philosophy

AI is used as an engineering assistant.

AI supports:

- implementation
- analysis
- documentation
- testing
- optimization
- problem solving

AI does not replace architectural decisions.

All generated solutions must follow the established project standards.

---

# 3. Core AI Rules

## 3.1 Respect Existing Architecture

AI must always understand existing:

- services
- modules
- entities
- APIs
- database structures

Before creating new functionality, AI must verify existing patterns.

---

## 3.2 No Uncontrolled Changes

AI must never:

- redesign architecture without approval
- remove existing systems
- create duplicate functionality
- introduce unnecessary technologies

---

## 3.3 Documentation Synchronization

Any significant change requires updating:

- architecture documentation
- API documentation
- database documentation
- feature documentation

Code and documentation must remain synchronized.

---

# 4. AI Development Workflow

Every AI-assisted implementation follows:

## Step 1

Understand requirements.

AI analyzes:

- requested feature
- existing architecture
- affected systems

---

## Step 2

Create implementation plan.

Plan includes:

- affected components
- required changes
- possible risks

---

## Step 3

Implement changes.

Implementation follows:

- coding standards
- architecture rules
- security requirements

---

## Step 4

Validate implementation.

Validation includes:

- tests
- error handling
- documentation

---

# 5. AI Code Generation Rules

Generated code must:

- follow project structure
- use existing patterns
- contain clear naming
- include validation
- support testing

Generated code must not:

- contain placeholders
- contain fake data
- create unnecessary abstractions

---

# 6. AI Understanding of Business Logic

AI must understand the domain before implementation.

Important concepts:

- skill-based competition
- fairness
- ranking integrity
- matchmaking
- progression
- player identity

Business rules have priority over technical convenience.

---

# 7. AI Database Rules

AI must respect:

- existing entities
- ownership boundaries
- migrations
- historical data

AI must not:

- modify production data directly
- delete competitive history
- create duplicate entities

---

# 8. AI API Development Rules

Before creating APIs, AI must define:

- endpoint purpose
- authentication requirements
- request format
- response format
- error handling

APIs must follow:

- versioning
- security standards
- documentation requirements

---

# 9. AI Frontend Development Rules

AI generated interfaces must follow:

- design system
- component reuse
- responsive behavior
- accessibility requirements

AI must avoid:

- inconsistent designs
- duplicated components
- hardcoded layouts

---

# 10. AI Backend Development Rules

Backend implementations must:

- keep business logic server-side
- validate all input
- handle errors correctly
- use proper service boundaries

---

# 11. AI Testing Requirements

AI generated features require tests.

Minimum:

- unit tests
- validation tests

Depending on feature:

- integration tests
- API tests
- performance tests

---

# 12. AI Security Requirements

AI must always consider:

- authentication
- authorization
- data protection
- abuse prevention

AI must never:

- expose secrets
- disable security checks
- trust client input

---

Development prompts should include:

- project context
- affected systems
- required behavior
- restrictions
- expected output

Example structure:

Project:

TheFastAndTheRich

---

# 14. AI Code Review

AI can assist with reviewing:

- bugs
- architecture problems
- security issues
- performance concerns

AI review does not replace human approval for critical changes.

---

# 15. AI Documentation Generation

AI may generate:

- technical documentation
- API descriptions
- test cases
- architecture updates

Generated documentation must reflect the actual implementation.

---

# 16. AI Agent Rules

Future AI agents working on the repository must:

1. Read project documentation first.
2. Understand existing architecture.
3. Create a plan before large changes.
4. Avoid unrelated modifications.
5. Update documentation after changes.
6. Run validation before completion.

---

# 17. AI Development Restrictions

AI must not:

- invent requirements
- create fake production data
- bypass security
- remove tests
- ignore existing standards
